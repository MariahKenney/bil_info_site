Get Your Brain Straight Hackathon

This page describes site-specific information about the Brain Image Library that hackathon participants may find useful. If your question is NOT answered here please check to see if it is answered on the main hackathon page.

BIL Helpdesk

The Brain Image Library has a helpdesk, that you should feel free to contact with any problems/issues/or questions. To submit a troubleticket, send email to: bil-support@psc.edu . It would be helpful to include that you are a hackathon participant in the email message and include the word "Hackathon" in the subject line.

Username/Password

If you requested an account, you should have received an email from grants@psc.edu with your username and instructions on how to change your password. All BIL systems use the same username and password. If you can not find the email or do not know what your username is, please email bil-support@psc.edu for assistance.

Hackathon Data

The BIL Filesystem, which is a lustre based filesystem, is mounted on all BIL systems at: /bil . The hackathon data itself can be found at: /bil/proj/hackathon/2022_GYBS/ Please read the README file in this directory for instructions on where to find the data and where to place your results.

BIL Hackathon Documentation

We highly recommend that participants review the BIL hackathon documentation. The BIL Hackathon documentation goes over several items to be aware of using the BIL analysis ecosystem, including how to connect using X2Go and connecting through a terminal program using ssh. If you haven't used them before, LMOD and SLURM are two importantitems to review and understand from the tutorial:
  • SLURM is an open source, fault-tolerant, and highly scalable cluster management and job scheduling system.
  • LMOD is software to manage software emvironments. It is used to make software available for use.
Note: There are a few sample SLURM scripts(aka jobs) that use LMOD Modules that you can examine and submit on the BIL analysis ecosystem at: /bil/proj/hackathon/2022_GYBS/BIL/slurm The hackathon documentation also goes into detail outlining several ways to containerize codes and also gives examples on how to run select software packages installed on the BIL system including MATLAB.

Analysis Ecosystem

The Brain Image Library analysis ecosystem provides an integrated computational and visualization system to explore, visualize, and and access data without having to download it. The underlying operating system runs Unix (CentOS). It consists of several parts including:

  • BIL Login Node: login.brainimagelibrary.org The BIL login node provides command line, secure shell (ssh) terminal access to BIL data. The login node is not intended for heavy computation or visualization, but rather to provide convenient shell access to BIL data and enable interactive and batch use of the BIL Computational Cluster.
  • BIL Workshop Node: workshop.brainimagelibrary.org The BIL Workshop node is a 56 multi-core VM node with about 2.6Tb of memory. It is intended to run desktop-visualization applications. It also acts as a front-end running SLURM and interact to enable interactive and batch use of the BIL Computational Cluster.
    • To run graphical applications, we recommend that you connect using X2Go The workshop node can be connected to using secure shell (ssh) for users that do not need to run graphical applications.
  • BIL Workshop GPU Node: workshop2.brainimagelibrary.org The BIL Workshop GPU node is a multi-core (72 physical, 144 virtual cores) machine containing two V100 (32Gb) GPUS and about 1.5Tb of memory. To run graphical applications that use the GPUs, we recommend that you connect using X2Go The workshop GPU node can also be connected to using secure shell (ssh) for users that do not need to run graphical applications.
  • BIL Computational Cluster: The BIL Computational cluster consists of eight nodes with 3 terabytes of RAM each. The computational cluster runs SLURM and is avaiable for both batch and interactive access from the BIL login node and the BIL Workshop Node. SLURM jobs can request a maximum of 75 cores per node
  • Other Systems: The BIL analysis ecosystem includes other machines that we will not be using for the hackathon including personalized VMs, the Bridges2 HPC system, and Neocortex a Cerebras advanced AI computing system. If you are interested in these systems, please see this page or contact bil-support@psc.edu
  • Data Transfer Nodes: upload.brainimagelibrary.org and download.brainimagelibrary.org These dedicated data transfer nodes can be used to move files to and from the BIL filesystem. There are many supported ways to upload and download files into your /bil home directory from the data transfer nodes, including rsync, Globus, sftp, and scp.
