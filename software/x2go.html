Software Available on BIL Systems: X2Go

Overview

X2Go is an open source remote desktop application that is installed on all BIL VM's. X2Go enables one to connect to and run traditional desktop visualization applications (such as Fiji - ImageJ) on a remote computing system.

This document outlines how to use the X2GO remote desktop application to access a virtual machine at BIL.

To use software that contains a gui, the X2Go software client should be installed locally, then used to connect to the BIL system.

If you have problems using the X2go software, do not hesitate to contact the BIL Helpdesk for assistance.

  1. Install X2Go client
  2. Connecting to a BIL VM (workshop.brainimagelibrary.org) using X2Go

Install X2Go client

Download and install the appropriate x2go client (Windows, Linux, or Mac) from: https://wiki.x2go.org/doku.php

Mac Users: You must also install xQuartz in order to run x2go: https://www.xquartz.org

Connecting to a BIL VM (workshop.brainimagelibrary.org) using X2Go

Below is an example using X2GO to connect to the BIL VM workshop.brainimagelibrary.org If you are connecting to a different BIL VM, substitute the name of that VM for workshop.brainimagelibrary.org in the instructions below.

  • Start the X2Go client software on your machine.
  • From the Session menu select New Session.
  • In the Session preferences Popup:
    • Set the Session Name to: workshop.brainimagelibrary.org
    • Under Server, set the Host to: workshop.brainimagelibrary.org
    • Under session type, set the selection to: MATE
    • Click the [OK] Button
  • Move the mouse cursor to the light gray box over the words: â€œviz.brainimagelibrary.org
  • click the LEFT mouse button
  • Enter your (login) username and your password then click the [OK] Button
  • After a few seconds, an X2GO desktop will appear on your screen
  • When you are done, be sure to select shutdown to completely terminate the X2Go session.


How to use specific software packages:

  1. Anaconda: Anaconda is installed as a module To run Anaconda:

    $ module load anaconda
    $ python3

    After loading the module, users can also run:
    • ipython
    • Jupyter Notebooks
    • Jupyter Labs
    Additional packages that are included in anaconda include: allensdk, biopython, bokeh, Dask, napari, nilearni, neuron, pandas, pytorch, scikit-image, scikit-learn, starfish, Theano. To see a full list of all available packages, type in the terminal window:

    $ pip list



  2. imagej-fiji: Fiji is installed as a module To run imagej/fiji:

    $ module load fiji $ fiji



  3. julia: Julia is installed as a module To run Julia:

    $ module load julia $ julia



  4. knime: Knime is installed as a module To run Knime:

    $ module load knime $ knime



  5. matlab: Matlab is installed as a module To run Matlab, users first need to fill out the following form: https://www.psc.edu/resources/software/matlab/permission-form/ After permission is granted,

    $ module load matlab $ matlab -options < input-file > output-file

    After loading the module, to see what toolkits are available:

    $ ver

    Additional documentation: http://www.mathworks.com/products/matlab/

  6. R and Rstudio:is installed as a module Interactive use of R and RStudio are available through Anaconda:

    $ module load anaconda $ conda activate rstudio



  7. scala: Scala is installed as a module To run Julia:

    $ module load scala $ scala



  8. vaa3d: Vaa3d is installed as a module
    9. To run Vaa3d:

    $ module load vaa3d $ vaa3d

Use of Pre-publication Data

The Library expects all users of the submitted data, unless explicitly granted by the data submitter or the data submitter's project, to grant the authors of the data the right to publish the first paper concerning the dataset within three years of deposit. If there is no publication associated with the dataset, please contact the listed authors for permission to publish. There is no restriction on post-publication data.
